Servers and Clients

Serverဆိုတာလည်း ကွန်ပျူတာပဲ။ ပိုကောင်းတဲ့ကွန်ပျူတာပေါ့။ အသုံးပြုသူအများကြီးကို အထောက်ပံ့ပေးနိုင်တယ်။ ရိုးရိုးကွန်ပျူတစ်လုံးကို serverအနေနဲ့သုံးမယ်ဆိုရင် user လေးငါးဆယ်လောက် လှမ်းပြီး request လှမ်းပို့တာနဲ့တင် စက်ကလေးသွားပြီး ရပ်သွားတာမျိုး သုံးမရလို့ ပိတ်ရ ပြန်ဖွင့်ရတာမျိုးလုပ်ရနိုင်တယ်၊ server သည်ရိုးရိုးကွန်ပျူတာထက် လုပ်ဆောင်နိုင်စွမ်းဘာလို့ ပိုကောင်းလဲဆိုတော့ သူ့မှာပါတဲ့ CPU နည်းပညာ၊ RAM၊ storageတွေသည် အဆပေါင်းများစွာပိုကောင်းလို့ဖြစ်တယ်

Clientဆိုတာကတော့ server ကိုလှမ်းဆက်သွယ်ပြီး web document သို့မဟုတ် တခြားသောလုပ်ဆောင်ချက်လုပ်ဖို့ လှမ်းချိတ်ရတဲ့ ကွန်ပျူတာကိုပြောတာဖြစ်တယ်။ အများစုကတော့ ရိုးရိုးကွန်ပျူတာတွေဖြစ်တယ်၊ သို့ပေမယ့် serverကို serverနောက်တစ်လုံးကလည်း Client အနေနဲ့ ချိတ်ဆက်ရတာမျိုးလည်းဖြစ်နိုင်တယ်

ဥပမာ

Googleကနေ serverတစ်ခုထားပြီး index တွေcrawlingလုပ်ထားပြီး search engine ကိုတည်ဆောက်ထားတယ်၊ ဆိုလိုတာက ဘယ်websiteမှာ ဘာရှိတယ် ဆိုတာကို မှတ်ထားတာမျိုးပေါ့၊ အဲ့တာကို clientဖြစ်တဲ့ computerရဲ့browserကနေ လှမ်းပြီးတော့ https://google.com/search?q=dog ဆိုပြီး dogအကြောင်းကို ရှာဖို့ request လှမ်းပို့ရတယ်၊ အဲ့တော့မှ serverဖြစ်တဲ့ search engineကနေ သက်ဆိုင်ရာ contentတွေကို browserဆီကိုပြန်ပို့ပေးတယ်။

ဒီနေရာမှာ serverနဲ့ clientအလုပ်လုပ်ပုံကိုပဲပြောချင်တာဖြစ်လို့ တခြားသုံးသွားတဲ့ web standardsတွေအကြောင်းထည့်မပြောထားဘူး။

အကျဉ်းချုပ်အနေနဲ့ server သည် request တွေကိုလက်ခံပြီးတော့ processလုပ်တယ် ရလာတဲ့ resultကို clientကို responseပြန်ပေးတယ်

clientသည် serverကို requestပို့တယ် serverကပြန်လာတဲ့responseကိုလက်ခံတယ်

Browser

Browserဆိုတာကတော့ websiteတွေ web applicationတွေကို ဖွင့်ဖို့သုံးတဲ့ကောင်လို့ အလွယ်ပြောလို့ရတယ်

တနည်းအားဖြင့် browserသည် clientတစ်ခုလည်းဖြစ်တယ်၊ view ကို renderလုပ်တဲံ့ engineတွေပါမယ်၊ ဆိုလိုတာက browserသည် UIပြဖို့အတွက် လည်း အလုပ်လုပ်တယ်၊ နောက်ထပ် အဲ့ဒီ UIတွေ တနည်းအားဖြင့် DOMတွေကိုထိန်းဖို့ programming languageဖြစ်တဲ့ javascriptအတွက် engineတွေလည်းပါတယ်
